独轮车插件做什么

装完进 B 站直播间，右下角 弹幕助手 按钮就是入口。它做两件事：

替你说 —— 独轮车循环 · 自动跟车 · 手动发送 + AI 润色 · 影子屏蔽自动改写。
替你看 —— Chatterbox Chat 接管右侧评论区 · 粉丝牌禁言巡检看今天哪些房间禁了你。

其它（智驾 / 烂梗库 / 同传 / LLM / 保安室跨房间同步）都是给这两件事打下手的。

民间自用/自制插件，不是 B 站官方功能。自动化（独轮车 / 自动跟车 / 智驾 / 自动重发）默认走试运行（只看不发），由你主动切到真发；请控制频率，别打扰主播和其他观众。

安装 B站独轮车插件

  1. 安装脚本管理器

    先安装 Tampermonkey 或 Violentmonkey。 Chrome / Edge 用户一般直接装 Tampermonkey，Firefox 用户也可以二选一。

  2. 打开 Greasy Fork

    进入 B站独轮车 + 自动跟车 Greasy Fork 发布页，点击安装，并在脚本管理器弹窗里确认。

  3. 打开 B 站直播间

    打开 B 站直播首页 或任意直播间，刷新页面后点击右下角 弹幕助手。

  4. 按需开启功能

    自动跟车、Chatterbox Chat、AI 规避、保安室同步和同传都是可选功能。

直播间独轮车主要功能

替你说是发送类工具（主线）；替你看是阅读类工具（辅助）。 其余 LLM / 烂梗库 / 同传 / 影子屏蔽自动学习等都在后台为这两件事服务。

替你说（发送类，主要）

独轮车循环发送

面向 B站直播间独轮车场景，多套模板、每行一句，支持固定/随机间隔、随机颜色、随机字符和长弹幕拆分。

自动跟车

识别短时间内被多人重复发送的弹幕，达到阈值后带冷却跟一句；三档预设 + 自定义；首次开启默认走试运行，由你主动切真发。

手动发送 + AI 润色

面板里手动一句弹幕。可选 AI 润色（用你自己的 prompt 让 LLM 改写后再发）；影子屏蔽时自动给改写候选。

替你看（阅读类，辅助）

Chatterbox Chat 评论区

接管右侧评论区，展示弹幕、礼物、醒目留言、舰队、进场等事件，支持偷弹幕、+1 和复制；深色模式自适应。

粉丝牌禁言巡检

一键看自己在哪些粉丝牌房间被禁言、屏蔽、风控或主播注销——重度观众的"健康检查"。

其余特性：智能辅助驾驶（智驾，LLM 选梗）、多源烂梗库（LAPLACE / SBHZM / chatterbox-cloud / 房间专属社区库）、同传翻译（Soniox）、LLM 多 provider（Anthropic / OpenAI / OpenAI 兼容自填 base URL）、影子屏蔽自动学习（后台默认开，不在设置里露面）、保安室 / 监控室代理跨房间同步（同一观众在 N 个直播间同开时聚合状态、跨标签接管 / 远程下发预设）。

权限说明

页面和网络范围（按用途分组）

脚本管理器在首次访问每个新域时仍会单独弹窗确认。不开对应功能，那一域就不会被访问。

  • 必要：@match *://live.bilibili.com/*（只在 B 站直播间运行）；chatterbox-cloud.aijc-eric.workers.dev（烂梗库默认聚合后端）；workers.vrp.moe（LAPLACE 替换规则 + 烂梗库源）；edge-workers.laplace.cn（LAPLACE AI 规避检查）。
  • 可选烂梗库 / 雷达源：sbhzm.cn（灰泽满直播间社区库）；live-meme-radar.aijc-eric.workers.dev（跨房间热门梗 🔥 徽章 + 可选趋势上报）。
  • LLM（填 key 才用）：api.anthropic.com / api.openai.com；@connect * 兜底让你能填 OpenAI 兼容自定义 base URL（DeepSeek / Moonshot / OpenRouter / Ollama / 小米 mimo 等）。
  • 同传：@require unpkg.com（Soniox 客户端）；api.soniox.com（识别 API）。
  • 本地开发 / 公会同步（普通用户用不到）：@connect localhost（本地后端测试）；bilibili-guard-room.vercel.app（保安室同步，后续将剥成姊妹脚本）。

脚本管理器能力

  • GM_addStyle：注入面板和评论区样式。
  • GM_getValue、GM_setValue、GM_deleteValue：本地保存设置、模板、规则和缓存。
  • GM_info：读取脚本元信息。
  • unsafeWindow：必要时与 B 站页面上下文交互。
  • @run-at document-start：尽早准备 UI、样式和聊天适配。

隐私和数据流说明

大部分数据只保存在你的浏览器里，包括弹幕模板、发送设置、自动跟车配置、替换规则、 Chatterbox Chat 偏好、粉丝牌巡检缓存和影子屏蔽观察记录。

B 站接口和 WebSocket

用于读取直播间事件、发送弹幕、获取当前账号相关粉丝牌房间信息和判断房间状态。

可选服务

Soniox 仅在启用同传/语音识别时使用；直播间保安室只在你开启同步时接收摘要或选定规则；烂梗库梗源（sbhzm.cn / chatterbox-cloud）仅在打开烂梗库或社区贡献时拉取列表；LLM provider（api.anthropic.com / api.openai.com 或你自填的 OpenAI 兼容 base URL）仅在填入 API key 并启用 AI 规避/改写时才调用。

托管资源

GitHub Pages 和 Greasy Fork 用于官网、安装和发布；unpkg 用于加载 Soniox 浏览器端客户端。

不要在 issue、截图或导出配置里公开 cookie、csrf token、账号密钥、localStorage dump、私人规则或私有同步地址。

常见问题和排障

  • 看不到按钮：确认脚本已安装并启用，然后刷新 B 站直播间页面。
  • 安装后没有更新：在脚本管理器里手动检查更新，或从 Greasy Fork 重新安装。
  • 弹幕发送失败：检查登录状态、直播间权限、弹幕长度、锁定表情权限和风控提示。
  • 自动跟车不触发：检查预设、阈值、冷却、唯一用户判断和融入黑名单。
  • Chatterbox Chat 空白：刷新页面，确认直播间有弹幕，临时关闭自定义 CSS。
  • 粉丝牌巡检未知：可能是登录状态、接口限流、主播注销、网络错误或 B 站接口变动，稍后重试。
  • 同传不可用：检查麦克风权限、浏览器支持，以及 unpkg/Soniox 资源能否加载。
  • 候选改写没有自动发送：默认“只给候选”模式不会自动重发，需要明确开启自动重发和 AI 规避。

已知限制

  • B 站页面结构、接口、风控策略和 WebSocket 数据格式可能随时变化。
  • 高流量直播间里，DOM 兜底解析可能漏掉部分事件。
  • 自动发送、自动跟车和自动重发都可能被 B 站限流、屏蔽或风控。
  • 影子屏蔽检测是启发式判断，不能百分百区分网络延迟、房间行为和真实屏蔽。
  • 替换规则和 AI 规避不能保证每条弹幕都能通过审核。
  • 同传准确率受麦克风质量、浏览器支持和 Soniox 服务可用性影响。
  • 主要测试路径是桌面浏览器加 Tampermonkey/Violentmonkey。
