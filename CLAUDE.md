# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Code style

- **Comment sparingly.** Do not add explanatory comments to every function/method. Skip any comment that just restates the function name, signature, or an obvious operation.
- Only add a comment when the logic is genuinely non-obvious — a subtle edge case, a non-obvious *why*, or a workaround — or when explicitly asked to document.
- Prefer self-documenting names over comments, and match the existing comment density of the file you're editing.
